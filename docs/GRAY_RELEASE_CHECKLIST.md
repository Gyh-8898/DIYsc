# 灰度发布检查清单（商业运营版）

## 1. 基础健康
- 后端服务可访问：`GET /api/health`
- 数据库连接正常（后台系统自检中显示“正常”）
- 支付配置、物流配置、七牛配置完整度已确认

## 2. 核心交易链路
- 小程序登录成功
- 创建订单成功（金额由后端重算）
- 发起支付成功（可拿到支付参数）
- 支付回调成功（订单状态进入“待发货/制作中”）
- 后台发货成功（填写物流公司和运单号）
- 前端订单详情可看到物流轨迹

## 3. 营销链路
- 优惠券：创建模板 -> 发放 -> 前端可见 -> 下单核销
- 积分：下单获取、抵扣、冻结、解冻流水一致
- 分销：佣金入账、提现申请、后台审核通过/拒绝

## 4. 风险与售后链路
- 投诉工单提交后，后台可查看详情并回复
- 用户端可看到回复记录与状态变化
- 取消订单、超时订单的库存释放正确

## 5. 运营管控链路
- 作品广场置顶/取消置顶生效，前端排序正确
- 管理后台关键操作有审计日志（配置、发货、优惠券、投诉、提现）
- 系统自检无“异常”项，警告项已确认可接受

## 6. 发布策略
- 先灰度（小流量）再全量
- 灰度窗口至少观察 30 分钟：订单成功率、支付成功率、投诉率
- 若异常，按审计日志和自检结果快速回滚
